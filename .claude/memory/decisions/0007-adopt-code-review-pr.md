Added an automated **PR code-review** that audits a GitHub pull request and **posts the result back as a PR comment** via `gh`. Chose to **adopt Anthropic's official `code-review` plugin engine** (the `/code-review` command by Boris Cherny, already installed + enabled in `~/.claude.json`) rather than build one from scratch or use it unmodified — a **hybrid**, mirroring [[0005-adopt-onredoak-design-review]]: keep its engine, widen its standards to FUKIONE's. Shipped as `.claude/commands/review-pr.md` → the `/review-pr [PR#]` slash command (registers as the `review-pr` skill). The prior state was only a *promise* of a review pass in [[git-flow-agent]] / [[0004-git-flow-trunk-based]] ("review is a separate pass, code-reviewer agent") with nothing wired up and nothing posting to GitHub.

**Why:** the official engine is genuinely good — eligibility gate (skip closed/draft/trivial/already-reviewed), 5 parallel review agents (standards / bug / git-history / prior-PR / code-comment), per-issue confidence scoring with a **≥80 filter** to kill false positives, and strict `blob/<full-sha>#Lx-Ly` links so comments render on GitHub. Rewriting that is wasted effort. But it only reads **`CLAUDE.md`** as its standard — it would miss FUKIONE's real rules, which live in `.claude/memory/conventions/` and the `CLAUDE.md` **Gotchas** (lead-saved-to-DB-first, prices in the Payload `Settings` collection, VN-no-accent URLs, no hand-editing Payload artifacts, secrets only in `.env`) and the **Testing** risk-priority. The wrapper injects those as explicit review criteria. Using it unmodified would grade FUKIONE by a generic checklist; building from scratch would re-derive the multi-agent + confidence orchestration for no gain.

**How to apply:**
- Run `/review-pr <PR#>` (or `/review-pr` to resolve the current branch's PR). Requires the `gh` CLI authenticated (already is, account `trinhvandat`); no MCP needed (unlike `/design-review`).
- It is the **separate review pass** the git-flow [[git-flow-agent]] mandates — run it from a context **other** than the one that authored the PR; never self-approve in the authoring context.
- Standards are the source of truth and the command loads them at review time: root `CLAUDE.md` (Gotchas + Scope/YAGNI), `.claude/memory/conventions/*`, and any relevant `.claude/memory/decisions/*`. UI/visual critique is deferred to `/design-review` ([[0005-adopt-onredoak-design-review]]), not duplicated here.
- It only posts when an issue clears **confidence ≥ 80**; otherwise it posts a short "no issues" comment or, if ineligible, stays silent and reports why. Comment + commit artifacts stay **English** per [[english-only-artifacts]].
- The always-apply trigger lives as the command's own description (auto-discovered); no `CLAUDE.md` edit was needed since this is a human-invoked command, not an always-on subagent rule — contrast [[where-to-codify-rules]].
